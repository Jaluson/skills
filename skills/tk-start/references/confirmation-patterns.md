# 确认模式参考

## 确认类型

### 1. 初步确认
在完成初始解析后，向用户确认是否理解正确。

### 2. 渐进确认
对于复杂任务，每完成一个阶段确认一次。

### 3. 最终确认
在开始执行前，获得用户的明确确认。

## 渐进式确认流程优化

### 多阶段确认机制

```
阶段 1：需求确认
    │
    ▼
┌─────────────────────────────────────┐
│  解析用户原始需求                    │
│  - 提取关键信息                      │
│  - 识别约束条件                      │
│  - 确认技术栈                        │
└─────────────────────────────────────┘
    │
    ▼
    确认：需求理解是否正确？

阶段 2：方案确认
    │
    ▼
┌─────────────────────────────────────┐
│  设计实现方案                        │
│  - 选择技术路径                      │
│  - 规划实现步骤                      │
│  - 评估风险点                        │
└─────────────────────────────────────┘
    │
    ▼
    确认：方案设计是否可行？

阶段 3：执行确认
    │
    ▼
┌─────────────────────────────────────┐
│  准备执行                            │
│  - 确认改动范围                      │
│  - 准备测试计划                      │
│  - 制定回滚方案                      │
└─────────────────────────────────────┘
    │
    ▼
    确认：是否可以开始执行？

阶段 4：结果确认
    │
    ▼
┌─────────────────────────────────────┐
│  验证执行结果                        │
│  - 检查交付物                        │
│  - 执行验证测试                      │
│  - 确认无回归问题                    │
└─────────────────────────────────────┘
    │
    ▼
    确认：结果是否满足需求？
```

### 确认频率控制

```
确认触发规则：
- 简单任务（< 5 分钟）：1 次确认（执行前）
- 中等任务（5-15 分钟）：2 次确认（方案 + 执行前）
- 复杂任务（> 15 分钟）：多次确认（每阶段一次）

确认冷却期：
- 同一类型确认在 2 分钟内不重复
- 除非用户主动询问
```

### 确认去重机制

```
当需要确认多个相似事项时：
❌ 错误做法：
确认 A
确认 B
确认 C
确认 D

✅ 正确做法：
合并为一个大类确认，或按优先级逐个确认
```

## 错误消息用户友好化

### 错误消息转换规则

```
原始错误格式：
Error: ENOENT: no such file or directory, open '/path/to/file'

用户友好格式：
❌ 找不到文件
文件路径：/path/to/file
建议：请检查文件是否存在，或路径是否正确
```

### 错误消息模板库

| 错误类型 | 用户友好消息 | 建议操作 |
|----------|--------------|----------|
| 文件不存在 | 找不到指定的文件 | 检查路径是否正确 |
| 权限不足 | 没有执行此操作的权限 | 联系管理员获取权限 |
| 网络超时 | 服务器响应超时 | 检查网络连接，稍后重试 |
| 编译错误 | 代码无法通过编译 | 查看错误详情并修复 |
| 依赖缺失 | 缺少必要的依赖 | 运行安装命令 |
| 资源不足 | 系统资源不足 | 关闭其他程序释放资源 |

### 错误消息展示格式

```
## ❌ 操作失败

**发生了什么**：无法保存文件
**为什么**：磁盘空间不足
**影响**：您的更改可能丢失

**您可以**：
1. [ ] 清理磁盘空间后重试
2. [ ] 保存到其他位置
3. [ ] 联系管理员

**详细信息**（可选查看）：
```
原始错误代码：ERR_DISK_FULL
可用空间：0.2 GB
需要空间：1.5 GB
```
```

## 确认时机

| 场景 | 确认类型 | 必须性 |
|------|----------|--------|
| 初始解析完成 | 初步确认 | 必须 |
| 发现多个可能方案 | 选择确认 | 必须 |
| 信息不完整 | 填空确认 | 必须 |
| 方案设计完成 | 最终确认 | 必须 |
| 子任务完成 | 阶段确认 | 建议 |
| 最终交付 | 交付确认 | 必须 |
