# 确认模式参考

## 确认类型

### 1. 初步确认
在完成初始解析后，向用户确认是否理解正确。

### 2. 渐进确认
对于复杂任务，每完成一个阶段确认一次。

### 3. 最终确认
在开始执行前，获得用户的明确确认。

## 确认话术模板

### 初步确认
```
## 🤔 初步理解

我理解为：
- 您想要：[核心需求]
- 涉及：[范围]
- 目标是：[期望结果]

**请确认是否正确？**
```

### 选择式确认
```
## 🤔 需要选择

您的需求可能是：

1. **[选项1描述]** → [处理方式]
2. **[选项2描述]** → [处理方式]
3. **[选项3描述]** → [处理方式]
4. **以上都不是** → 请补充说明

请输入选项编号（1-4）：
```

### 填空式确认
```
## 🤔 需要补充

以下信息还需要确认：

| 项目 | 您的回答 |
|------|----------|
| 涉及模块 | __________ |
| 时间要求 | __________ |
| 优先级 | __________ |
| 特殊要求 | __________ |

请填写：
```

### 确认清单
```
## ✅ 执行前确认

请确认以下清单：

- [ ] 任务目标：[描述]
- [ ] 涉及范围：[描述]
- [ ] 交付标准：[描述]
- [ ] 时间要求：[描述]

**输入 "确认" 开始执行，或指出需要修改的部分：**
```

## 用户反馈处理

### 收到确认
```
收到确认，开始执行任务。
```

### 收到修改意见
```
好的，根据您的反馈：
- 修改项1
- 修改项2

已更新为：
[更新后的内容]

**请再次确认：**
```

### 收到新信息
```
感谢补充新信息，正在重新解析...

[重新展示解析结果]
```

## 确认时机

| 场景 | 确认类型 | 必须性 |
|------|----------|--------|
| 初始解析完成 | 初步确认 | 必须 |
| 发现多个可能方案 | 选择确认 | 必须 |
| 信息不完整 | 填空确认 | 必须 |
| 方案设计完成 | 最终确认 | 必须 |
| 子任务完成 | 阶段确认 | 建议 |
| 最终交付 | 交付确认 | 必须 |
