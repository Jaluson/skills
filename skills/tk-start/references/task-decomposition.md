# 任务拆解方法论

## 拆解原则

### 1. MECE 原则
- **相互独立**：各子任务不重叠
- **完全穷尽**：覆盖所有必要方面

### 2. 渐进式拆解
先大后小，先粗后细

### 3. 可执行性原则
每个子任务应可独立执行和验收

## 拆解维度

### 目标维度
- 最终交付物是什么？
- 成功的衡量标准是什么？
- 需要达到的质量水平？

### 范围维度
- 涉及哪些技术栈？
- 涉及哪些模块/组件？
- 是否有边界限制？

### 资源维度
- 可用的技术资源？
- 时间限制？
- 人员限制？

### 风险维度
- 已知的技术难点？
- 可能的风险点？
- 需要的外部依赖？

## 问题发现清单

当遇到以下情况时，必须标记为"需确认"：

- [ ] 任务目标模糊或有多重含义
- [ ] 范围边界不清晰
- [ ] 缺少必要的上下文
- [ ] 存在歧义的术语
- [ ] 时间要求不明确
- [ ] 优先级不明确
- [ ] 质量标准不明确
- [ ] 假设条件未验证
