# 调度报告增强指南

## 概述

本文档定义 tk-start 调度器的报告增强机制，提供更丰富的任务执行摘要、资源消耗分析、风险识别和后续建议。

## 任务执行摘要生成

### 摘要生成时机

```
1. 任务完成后自动生成
2. 用户请求时生成
3. 异常发生后生成
4. 定时生成（每 5 分钟）
```


## 风险点识别与提示

### 自动风险识别

```
风险识别时机：
1. 任务启动时
2. 任务执行中（每 5 分钟）
3. 任务完成前
4. 异常发生时
```

### 风险分类

| 风险等级 | 符号 | 说明 | 处理方式 |
|----------|------|------|----------|
| 严重 | 🔴 | 阻塞任务执行 | 立即处理 |
| 高 | 🟠 | 可能影响结果 | 重点关注 |
| 中 | 🟡 | 潜在问题 | 监控 |
| 低 | 🟢 | 可忽略 | 记录 |

### 风险识别场景

```
## 🚨 风险识别报告

### 当前风险
| 风险 | 等级 | 位置 | 影响 |
|------|------|------|------|
| 上下文使用超 80% | 🟠 高 | 全局 | 可能导致任务中断 |
| Agent-3 执行超时常 | 🟡 中 | 前端组件 | 整体进度延迟 |
| 依赖链过长 | 🟡 中 | Agent-2→3→4 | 一环失败影响大 |

### 历史风险统计
| 风险类型 | 出现次数 | 处理方式 |
|----------|----------|----------|
| 上下文不足 | 3 次 | 压缩策略调整 |
| 超时 | 2 次 | 超时阈值优化 |
| 依赖异常 | 1 次 | 重试成功 |

### 风险缓解建议
1. 🔧 增加上下文压缩频率
2. 🔧 拆分长任务为短任务
3. 🔧 建立依赖检查机制
```
